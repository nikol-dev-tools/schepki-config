## 1. Суть проекта
Пользователь (начинающий разработчик) создает портфель из 30+ простых, атомарных браузерных расширений («щепок») для US рынка.
* **Монетизация:** Донаты (Buy Me a Coffee) + потенциально подписки/реклама в будущем.
* **Инфраструктура:** Единая система управления (Remote Config) для всех расширений.
* **Аналитика:** Единый счетчик GA4 для отслеживания всех установок и активности.
* **Технологии:** Только Vanilla JS, HTML, CSS (Manifest V3). Никаких сложных фреймворков (React/Vue).

## 5. Инструкция для ИИ (Как работать в новых диалогах)
1. **Всегда используй `schepki-boilerplate`** при создании новых расширений. Клонируй его, меняй название, иконки и основную логику.
2. **Не трогай логику Remote Config и GA4** в бойлерплейте — она уже работает идеально.
3. **Обновляй статусы:** Если пользователь говорит, что расширение одобрили — обнови `README.md` в репозитории `schepki-config` (поменяй 🟡 на 🟢).
4. **Стиль общения:** Пользователь — новичок. Объясняй всё просто, пошагово, без сложного технического жаргона.
5. **Публикация:** При подготовке к публикации всегда генерируй промо-материалы (иконки 128x128, скриншоты 1280x800, промо-баннеры 440x280) и пиши SEO-оптимизированное описание для US рынка.
